# PRDs

## Purpose
Product Requirement Documents for the Marketplace Experience.

---

## Naming Convention

```
[feature-name]-prd.md
```

Files live **flat** in this folder (no product-area subfolders). Use a descriptive
feature name, e.g. `connection-request-flow-prd.md`, `search-relevance-prd.md`.
Register each PRD in `../../feature-index.yaml` so related RFCs, metrics, and Jira
keys are discoverable in one place.

## Creating New PRDs

Use the `/prd` skill to draft a PRD. It will guide you through the required
sections and apply our writing voice.

---

## PRD Template Sections

1. **Overview** — Problem statement, goals, success metrics
2. **User Stories** — Who benefits and how
3. **Requirements** — Functional and non-functional
4. **Design** — UX flows, Figma links
5. **Technical Considerations** — Architecture, dependencies
6. **Launch Plan** — Rollout strategy, flags, metrics
