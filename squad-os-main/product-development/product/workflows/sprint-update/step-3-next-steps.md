# Step 3: Next Steps

## Goal

Produce the forward-looking section: upcoming priorities, blockers, and decisions needed.

---

## Process

### 1. Draft upcoming priorities
Based on the sprint status (Step 1), the roadmap, and OKRs, draft the top priorities for the next sprint — framed by objective (Demand / Density).

### 2. Surface blockers & decisions
Pull anything blocking progress (dependencies on other teams — e.g. Data Foundations — external factors, open questions) and any decisions the squad or stakeholders need to make.

### 3. Interactive check
Prompt: **"What are the top priorities for next sprint? Any blockers or decisions I should flag to stakeholders?"**

### 4. Write to output file
Append below the metrics snapshot.

---

## Output format

```markdown
## Next Steps

### Priorities (next sprint)
- **[Priority]** — [why / expected outcome]

### Blockers & decisions needed
- **[Blocker/decision]** — [owner / what's needed]
```

## Notes
- Keep priorities outcome-framed, not just a task list.
- Call out cross-team dependencies explicitly (especially Data Foundations for metrics/instrumentation).
