# Step 5: Review & Finalization

## Goal

Final review of the complete sprint update. Ensure it's accurate, well-framed, and ready to share.

---

## Process

### 1. Read back the full file
Read `output/YYYY-MM-DD-sprint-update.md` and summarize what's included (sections present, anything carried forward, whether the Google Doc push succeeded).

### 2. Ask for revisions
Prompt: **"Any sections to revise? Any ad-hoc content to add (a strategic note, a call-out for stakeholders)?"**

Make edits in the markdown file, present updated sections, and re-push to Google Docs if needed.

### 3. Final review checklist
- [ ] **Accuracy:** sprint statuses reflect reality, not just Jira
- [ ] **Framing:** headings are outcome/goal-led, not feature-led
- [ ] **Metrics:** numbers sourced correctly; anything off-track is flagged
- [ ] **Next steps:** priorities are outcome-framed; blockers have owners
- [ ] **Formatting:** tables render, no broken markdown
- [ ] **Google Doc:** (if used) pushed and formatted correctly

### 4. Sign-off
Prompt: **"Everything look good? Ready to share?"** Confirm the Google Doc link is shareable with the intended audience and note any follow-ups.

---

## Notes
- The markdown file in `output/` is the canonical source and the historical archive.
- Skipping the Google Doc push and sharing manually is always fine.
- Mark ad-hoc additions clearly as one-offs, not part of the recurring structure.
