# Engineering

Engineering plans, RFCs, and bug investigations for the Marketplace Experience.

## Folders

| Folder | What's Here |
|--------|-------------|
| `plans/` | Implementation plans for upcoming features |
| `rfcs/` | Technical design proposals and architecture decisions |
| `bug-investigations/` | Dated investigation plans for production bugs |

Organized **flat** (no product-area subfolders). Tie engineering artifacts to features via `../feature-index.yaml`.

> Subfolders are created as content lands.

## Naming Conventions

- **Plans:** `{feature-name}.md` (e.g., `connection-request-flow.md`)
- **RFCs:** `{feature-name}-rfc.md`
- **Bug investigations:** `bug-{MM-DD-YYYY}-{description}/investigation-plan.md`
